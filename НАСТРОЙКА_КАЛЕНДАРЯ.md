# Подключение Google Календаря к боту

Когда календарь подключён:

- **Запись ученика → урок появляется в твоём Google Календаре** (с именем ученика).
- **Отмена / перенос урока → событие в календаре тоже убирается/переезжает.**
- **Бот не предлагает ученикам время, когда ты занята** — он смотрит твой календарь и прячет пересекающиеся слоты.

Если не настраивать — бот работает как раньше, просто без календаря. Ничего не сломается.

Настройка делается один раз, минут на 15. Дальше всё автоматически.

---

## Идея простыми словами

Боту нужен «робот-помощник» — это **сервисный аккаунт** Google. Ты создаёшь такого робота, даёшь ему доступ к своему календарю (как будто делишься календарём с коллегой), и бот от его имени добавляет уроки. Браузер и пароли при этом не нужны — поэтому это работает на хостинге.

---

## Шаг 1. Создать проект и включить Calendar API

1. Зайди на **https://console.cloud.google.com** под своим Google-аккаунтом.
2. Вверху нажми на выбор проекта → **New Project** (Новый проект). Назови как угодно, например `tutor-bot`. Создай.
3. Убедись, что вверху выбран именно этот проект.
4. В поиске вверху набери **Google Calendar API** → открой → нажми **Enable** (Включить).

## Шаг 2. Создать сервисный аккаунт («робота»)

1. Слева меню → **APIs & Services** → **Credentials** (Учётные данные).
2. **Create Credentials** → **Service account**.
3. Имя, например `tutor-bot-calendar`. Нажми **Create and Continue**, затем **Done** (роли можно не назначать).

## Шаг 3. Скачать ключ (JSON)

1. В списке **Service Accounts** нажми на только что созданный.
2. Вкладка **Keys** → **Add Key** → **Create new key** → тип **JSON** → **Create**.
3. Браузер скачает файл вроде `tutor-bot-calendar-xxxx.json`. **Храни его как пароль — никому не показывай.**
4. Открой этот файл и скопируй из него строку **`client_email`** — это email робота, вида
   `tutor-bot-calendar@tutor-bot.iam.gserviceaccount.com`. Понадобится в шаге 4.

## Шаг 4. Дать роботу доступ к своему календарю

1. Открой **https://calendar.google.com**.
2. Слева наведись на нужный календарь (обычно он называется твоим именем) → три точки → **Settings and sharing** (Настройки и доступ).
3. Раздел **Share with specific people** → **Add people** → вставь email робота из шага 3.
4. Уровень доступа выбери **Make changes to events** (Вносить изменения в мероприятия). Сохрани.
5. На этой же странице ниже найди **Calendar ID** (Идентификатор календаря). Для основного календаря это обычно твой email (`...@gmail.com`). Скопируй его — это `GOOGLE_CALENDAR_ID`.

## Шаг 5. Прописать настройки боту

Дальше зависит от того, где запускается бот.

### Если бот на компьютере (локально)

1. Положи скачанный JSON-файл в папку с ботом и **переименуй его в** `service_account.json`.
2. В файле `.env` добавь строку с твоим Calendar ID:
   ```
   GOOGLE_CALENDAR_ID=твой_email@gmail.com
   ```
3. Установи новые библиотеки и перезапусти бота:
   ```
   pip install -r requirements.txt
   python main.py
   ```

### Если бот на хостинге (Amvera)

⚠️ Важно: Amvera **не разрешает кавычки** в значении переменной, а в JSON их полно.
Поэтому JSON надо сначала превратить в base64 (одну длинную строку из букв и цифр без кавычек).
Бот сам распознаёт и обычный JSON, и base64 — так что для Amvera используем base64.

1. На своём Mac открой **Терминал** и выполни (подставь имя скачанного файла):
   ```
   base64 -i ~/Downloads/tutor-bot-XXXX.json | tr -d '\n' | pbcopy
   ```
   Это закодирует ключ и **скопирует результат в буфер обмена**.
2. В панели Amvera → твой проект → **Переменные окружения** добавь две переменные:
   - `GOOGLE_CALENDAR_ID` = **длинный адрес календаря** вида
     `3a0e3ac7...@group.calendar.google.com` (это «Идентификатор календаря» из настроек
     календаря — НЕ email робота!)
   - `GOOGLE_CREDENTIALS_JSON` = вставь сюда строку из буфера (Cmd+V). Галочку «Это секрет»
     можно поставить.
3. Сохрани и перезапусти бота (новый деплой).

> Содержимое ключа — секрет. В переменных окружения хранить его безопасно, в коде/гите — нельзя
> (файл `service_account.json` уже добавлен в `.gitignore`).

**Частая ошибка:** в `GOOGLE_CALENDAR_ID` случайно вписывают email сервисного аккаунта
(`tutor-bot-calendar@...iam.gserviceaccount.com`). Это неверно. Туда идёт **адрес календаря**
(`...@group.calendar.google.com` или твой `@gmail.com`, если пишешь в основной календарь).
Email робота нужен только для того, чтобы расшарить ему календарь (шаг 4).

---

## Как проверить, что заработало

1. В логах бота при старте появится строка `Google Calendar: подключение готово`.
2. Запиши тестового ученика на любой слот (или сделай это сама через бота) → в твоём Google Календаре должно появиться событие «🇬🇧 Урок английского — Имя».
3. Отмени эту запись → событие из календаря исчезнет.
4. Поставь в календаре своё событие на какое-то время и попробуй показать ученику свободные слоты — это время не предложится.

## Если что-то не работает

- **В логах нет «подключение готово»** → проверь, что переменные заданы и JSON скопирован целиком (с `{` в начале и `}` в конце).
- **Событие не создаётся** → чаще всего роботу не дали доступ к календарю (шаг 4) или указан не тот Calendar ID.
- **Доступ «Make changes to events»** обязателен — права «See all event details» недостаточно для создания событий.
- Любые сбои календаря бот переживает спокойно: запись/отмена в самом боте всё равно сработают, просто без отражения в календаре.

## Настройки (.env)

| Переменная | Что это | Обязательна |
|---|---|---|
| `GOOGLE_CALENDAR_ID` | Календарь, куда писать уроки (обычно твой email) | да, для включения |
| `GOOGLE_CREDENTIALS_JSON` | Ключ робота строкой (для хостинга) | одно из двух |
| `GOOGLE_CREDENTIALS_FILE` | Путь к файлу ключа (по умолчанию `service_account.json`) | одно из двух |
| `LESSON_DURATION_MIN` | Длительность урока в минутах (по умолчанию 60) | нет |
